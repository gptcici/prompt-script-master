
# Character Blocking Contract 多人物调度与稳定合约

## 目的

本规则用于多人镜头、群像、舞台表演、对话、角色站位、人物关系、手脸稳定和群演动态控制。它解决“所有人都乱动”“主次不清”“手脸崩坏”“多人镜头动作冲突”等问题。

## 启动条件

出现以下任一情况时读取本规则：

```text
两人以上、多人、群像、舞台、伴舞、群演、对话、对峙、主角和配角、前景人物和背景人物、众人反应、角色站位、多人动作、多人镜头、人物关系
```

## 角色标签规则

两个以上人物出现时，必须建立稳定标签。不要用含糊的“他们 / 她们”让模型自行分配动作。

可用标签：

```text
女主、男主、侍女A、舞者B、背景人群、前景歌手、图1人物、图2人物、Character A、Character B
```

每个主要人物至少明确：

```text
标签、位置、身份/服装锁定、动作、表情、视线、当前情绪、不可改变项。
```

## 三层动作层级

多人镜头中，每个可见人物只能归入一个动作层级。

### 1. 持续微动作层

默认给非焦点人物。

```text
呼吸、眨眼、肩膀轻微起伏、发丝轻动、视线轻微移动、站姿保持。
```

### 2. 单一反应层

只给当前镜头的焦点人物或焦点配角。

```text
女主抬眼半秒后收回视线。
男主握紧杯沿后放下。
舞者B在副歌第一拍转头并停住。
```

### 3. 大动作层

默认禁止，除非它就是这一镜头的唯一核心动作。

```text
站起、走动、转身、奔跑、拥抱、递物、打斗、多人交叉走位。
```

若必须使用大动作，同一镜头只允许一个焦点大动作，其他人物保持微动作。

## 多人调度规则

1. 先确定英雄主体，其他人物降低动作复杂度。
2. 每个人的动作必须分配，不让模型猜。
3. 接触动作必须写清接触点和终点。
4. 群演只写整体节奏，不逐个写面部细节。
5. 多人对话优先稳定机位和眼线，避免同时大幅转身。
6. 舞台群舞优先写队形、节拍、主角动作和背景舞者低幅同步。

## 手脸稳定规则

多人镜头中，手和脸最容易崩坏：

- 避免多人同时做复杂手部动作。
- 避免脸部被手、袖子、道具频繁遮挡。
- 对口型角色保持镜头稳定和头部低幅运动。
- 背景人物不要求精细五官，只保留群体轮廓和节奏。
- 用道具、光线、站位表现关系，不全靠脸部微表情。

## 与人物镜头强制覆盖规则的关系

本规则不替代人物镜头强制覆盖规则。它只负责多人镜头的动作分配和稳定性。

执行顺序：

```text
先确定人物镜头触发项
再用本规则分配多人动作层级
再写表情、肢体、口型、面光和动态材质
```

## 负面提示词建议

```text
人物类：多人脸部漂移、背景人物五官抢戏、主次混乱、群演动作过度、多人同时大幅转身、多人手部崩坏、人物互相穿插、眼线不一致
```
