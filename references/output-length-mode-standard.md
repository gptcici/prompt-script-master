# 输出长度档位与模式优先标准（V0.9.82）

本文件用于控制最终提示词长度、输入模式和输出格式，避免所有任务都写成同一种长视频提示词。

## 核心原则

1. 先判断模式，再决定格式和长度。
2. 视频提示词继续使用 `【正文提示词】 + 【负面提示词】`。
3. 图片提示词默认使用自然语言段落，不强制拆正负面，不输出视频时间轴。
4. 图生视频只写图片中看不出来的内容：动作、镜头、光线变化、动力学、声音和保持约束；但前提是参考图关键视觉信息清晰可识别。
5. 提示词不是越长越好。长度由任务复杂度、参考图数量、镜头数量、锁脸难度、动作密度和平台输入预算共同决定。

## 模式优先判断

收到任务后，在 `S1_INPUT_RECOGNITION` 与 `S2_REFERENCE_SCAN` 阶段内部先判断以下模式：

- `T2V`：纯文字生成视频。需要写完整主体、动作、场景、镜头、光影、动力学和约束。
- `I2V`：单图生成视频。先用 `reference-visibility-gate.md` 判断图片中的人物、服装、背景和构图是否清晰；清晰部分不重复展开，只补动作、运镜、光线变化、动力学和保持项；不清晰或异常部分必须补写或重建。
- `R2V`：多参考图 / 视频 / 音频混合。必须先执行参考图职责转移契约，说明每个素材只控制什么、不控制什么。
- `FLF2V`：首帧尾帧过渡。首帧控制起点，尾帧控制终点，中间只写连续运动路径，不提前抵达尾帧。
- `V2V / Edit / Extend`：视频参考、局部编辑或续写。只写需要改变或延续的变量，不重建全部画面。
- `IMAGE`：图片生成 / 重绘 / 角色图 / 海报 / 三视图。默认自然语言图片提示词，不写视频时间轴。

## 长度档位

### A. 压缩版

适用：用户要求“简短 / 压缩 / 一句话 / 快速版本”。

- 中文约 80-180 字。
- 只保留主体、动作、场景、一个镜头意图、一个光影锚点、一个约束。
- 不写完整多镜头时间轴。

### B. 快速视频版

适用：简单单镜头或 5-8 秒视频。

- 中文约 120-260 字。
- 通常 1 个核心镜头，最多 2 个镜头。
- 适合 T2V 快速草案、I2V 轻动作、产品或人物小动作。

### C. 标准视频版

适用：普通 8-15 秒视频、轻中等复杂度 MV、人物情绪、单角色动作。

- 中文约 500-1000 字。
- 通常 2-3 个镜头。
- 每个镜头只保留一个主动作、一个主运镜、一个状态变化。

### D. 高控制视频版

适用：多参考图、锁脸换装、古风角色、首尾帧、复杂光影、复杂动力学、商业广告或强连续性任务。

- 中文约 1000-1600 字。
- 通常 2-4 个镜头。
- 必须保留参考图职责、全局光源锚点、全局物理动力学锚点、镜头时间轴和全局一致性。

### E. 密集分镜 / 导演说明版

适用：用户明确要求详细分镜、20宫格、工业故事板、分镜脚本、拍摄方案。

- 可超过 1600 字。
- 只在用户明确需要“详细分镜 / storyboard / shot list / 拍摄方案”时使用。
- 不作为默认视频提示词长度。

### F. 图片提示词长度

- 普通图片：中文约 150-450 字。
- 角色设定图 / 海报 / 精细风格：中文约 450-900 字。
- 三视图 / 多视角 / 复杂参考图融合：可更长，但不输出视频时间轴。
- 图片提示词默认不拆【负面提示词】，除非用户要求、目标平台支持，或任务是提示词审查。

## V0.9.81 参考图可识别性前置条件

I2V 极简原则不是无条件省略。执行“只写图片看不出来的东西”之前，必须先完成参考图可识别性检查：

- 图片里的人脸、服装、发型、背景、构图清晰：可以省略重复描述。
- 某一项不清晰：该项必须补写，例如服装被裁切就补写服装结构，背景死黑就补写场景空间。
- 人脸存在畸形、五官错位、异常色块、黑斑、异常阴影、大面积黑色、皮肤质感不符合生理逻辑：不得作为身份细节继承，必须触发 `face-integrity-check.md`，正文写自然面部结构和皮肤光影，负面提示词写对应故障项。

## I2V 极简原则

图生视频在参考图关键视觉信息清晰时遵循：`只写图片看不出来的东西`。如果参考图不清晰或存在异常，必须补写不清晰部分，不得盲目省略。

图片已经明确呈现的内容包括：人物脸、服装、发型、背景、构图、道具、色调和主体位置。最终提示词只需要强调：

- 保持哪些内容不变。
- 动作如何开始和结束。
- 镜头如何移动或保持稳定。
- 光线、风、雨、烟、水面或其他物理动态如何变化。
- 哪些内容不能漂移。

错误：重新长篇描述图片里的脸、衣服、背景，导致模型重新设计主体。

正确：图1作为首帧，严格保持人物身份、服装、发型和亭内构图；只让她右手缓慢拨弦，镜头轻微推近，夕阳侧逆光增强，纱幔和发带沿同一风向轻动。

## 负面与约束转换

外部工作流常用 `Constraints / Do not / Preserve / Change only`。在本 Skill 中统一转换为：

- 正向保持项写入正文，例如“保持人物身份、服装结构、首帧构图不变”。
- 故障和禁止项写入【负面提示词】。
- 动态故障统一放入“动力学类”，包括风向混乱、烟雾无来源、水面无接触、粒子无落点、道具漂浮、碰撞无反作用。

## 与其他规则的关系

- 本文件只决定模式和长度，不替代人物镜头强制覆盖规则。
- 复杂人物镜头仍须读取对应的人物、面光、动作、动力学、镜头语言和参考图规则。
- 如果本文件的长度档位与旧模板冲突，以本文件为准。


## V0.9.82 参考图重生建议前置条件

I2V 极简原则必须排在参考图上传后自检之后。若参考图多处不清晰，或用户指定的主职责区域不可辨，应优先建议用户重新生成 / 上传更清晰的参考图，而不是继续压缩提示词。

判断顺序：

```text
上传参考图
  ↓
reference-intake-self-check.md
  ↓
reference-visibility-gate.md + face-integrity-check.md
  ↓
可直接用 / 部分可用需补写 / 异常需重建 / 建议重新生成参考图
  ↓
再决定是否执行 I2V 极简原则
```

当进入“建议重新生成参考图”状态时，最终回答应说明：哪几类信息不清晰、为什么会影响锁脸/换装/场景/首尾帧，以及新参考图需要满足什么条件。
