# Reference Transfer Contract 参考图职责转移契约

## 目的

参考图职责转移契约用于多参考图、多图融合、锁脸换装、角色定妆图、服装图、发饰图、场景图、构图图、首帧、尾帧和关键帧任务。

它解决的问题：

- 多张参考图职责混乱。
- 用服装图时误污染人物脸。
- 用人物图时误继承错误服装。
- 场景图影响人物身份。
- 动作参考图改变角色长相。
- 风格图覆盖角色定妆。
- 首帧 / 尾帧职责不清导致过渡失败。

## 启动条件

只要满足以下任一条件，就启动本规则：

```text
用户提供两张及以上参考图
用户说图1的脸、图2的衣服、图3的发饰、图4的场景
用户要求锁脸、换装、多图融合、参考图反推
用户提供人物定妆图、角色模板、三视图、多视角
用户提供首帧、尾帧、关键帧
用户提供构图参考、动作参考、风格参考、场景参考
```

## 状态机挂接位置

本规则在以下阶段启动：

```text
S2_REFERENCE_SCAN
S6_BRANCH_CHECK
S8_FINAL_GENERATION
S9_INTERNAL_QA
```

- `S2_REFERENCE_SCAN`：识别每张参考图的主职责。
- `S6_BRANCH_CHECK`：判断是否触发锁脸、服装、场景、首尾帧、多图融合分支。
- `S8_FINAL_GENERATION`：在最终提示词中写清参考图职责。
- `S9_INTERNAL_QA`：检查是否发生参考图污染。

## V0.9.82 上传后自检与可识别性前置检查

在分配参考图职责之前，必须先读取 `reference-intake-self-check.md` 与 `reference-visibility-gate.md` 判断每张参考图的关键信息是否清晰可识别。职责只能分配给清晰可信的信息；当参考图不清晰区域较多或主职责区域不可辨时，应建议用户重新生成参考图，而不是强行分配高权重职责。

- 人脸清晰可信：可作为 `identity` 职责。
- 人脸模糊、遮挡、过暗、过曝或比例不可辨：不能只写“锁脸”，必须补写脸型、五官气质、年龄感、妆容和发型等身份锚点。
- 人脸存在畸形、五官错位、异常色块、黑斑、异常阴影、大面积死黑或皮肤质感不符合生理逻辑：不得继承异常内容，必须同时读取 `face-integrity-check.md`，只抽象继承可用身份信息。
- 服装、发饰、背景或构图不清晰：对应职责不能只写“保持图中内容”，必须在最终提示词中补写可见结构。
- 参考图多处不清晰，或用户指定的主职责区域不可辨：该图不得承担强职责，应降级为弱参考，并建议用户重新生成 / 上传更清晰的参考图。


## 参考图职责类型

每张参考图必须优先归入一个主职责。必要时可以有一个次职责，但不得无限扩展。

```text
identity：人物身份 / 脸 / 五官 / 年龄感 / 气质
hairstyle：发型 / 发色 / 发量 / 发饰位置
costume：服装结构 / 廓形 / 材质 / 分层 / 纹样
accessory：头饰 / 耳饰 / 项链 / 腰饰 / 武器 / 手持物
pose：姿态 / 动作 / 身体方向 / 手势
expression：表情 / 眼神 / 情绪状态
composition：构图 / 景别 / 画面比例 / 主体位置
environment：场景 / 建筑 / 空间关系 / 地形
lighting：光源方向 / 色温 / 明暗结构 / 氛围
style：整体风格 / 色彩情绪 / 摄影或绘画质感
first_frame：首帧 / 当前片段开始状态
last_frame：尾帧 / 当前片段结束状态
motion：动态趋势 / 运镜 / 动作节奏
```

## 最终提示词写法

最终提示词中的参考图职责要清晰写成：

```text
参考图职责：图1仅用于锁定女主脸型比例、五官气质、眼神感觉、鼻唇印象、年龄感和身份气质，不继承图1服装和背景。图2仅用于参考服装廓形、抹胸平口结构、布料分层、刺绣纹样和披帛材质，不继承图2人物脸。图3仅用于参考长发带、发饰材质和发型装饰逻辑，不改变图1人物身份。图4仅用于参考亭子空间、构图层次和山间云雾环境，不控制人物长相、服装和动作。
```

## 多图优先级

默认优先级如下：

```text
用户明确指令
> 人物身份参考图
> 首帧 / 尾帧状态参考
> 服装参考
> 发型 / 发饰参考
> 姿态 / 动作参考
> 场景 / 构图参考
> 灯光 / 风格参考
```

如果用户明确指定优先级，以用户指令为最高优先级。

## 职责隔离规则

1. 身份图只控制人物身份，不自动控制服装、场景、构图。
2. 服装图只控制服装，不控制脸。
3. 发型 / 发饰图只控制头发和饰品，不控制脸和服装，除非用户明确要求。
4. 场景图只控制环境、建筑、空间和氛围，不控制人物。
5. 构图图只控制画面布局、景别、主体位置和透视，不控制人物身份。
6. 风格图只控制色彩、质感、情绪，不得覆盖人物身份。
7. 首帧控制当前片段起点，尾帧控制当前片段终点。
8. 动作参考图只控制动作意图，必须服从当前人物身份和真实人体结构。
9. 多图冲突时，优先保护人物身份和用户明确指令。
10. 不得把所有参考图平均混合成一个新角色。

## 常见场景

### 锁脸换装

用户说：

```text
让图1人物穿上图2衣服。
```

内部职责：

```text
图1：identity + hairstyle
图2：costume
```

最终写法：

```text
以图1作为人物脸部、发型、年龄感和身份气质强参考；图2只作为服装廓形、材质、层次和纹样参考。生成结果必须保持图1同一个人物，不得继承图2人物脸。
```

### 图1脸 + 图2服装 + 图3发饰 + 图4场景

最终写法：

```text
图1用于锁定人物脸型、五官、眼神、鼻唇印象、年龄感和气质；图2用于锁定服装结构、布料层次、胸口形状、袖口和裙摆；图3用于锁定发饰、发带、发色细节和头发装饰方式；图4用于锁定场景空间、亭子结构、山雾层次和构图关系。各参考图职责分离，不得互相污染。
```

### 首帧 / 尾帧

最终写法：

```text
首帧用于锁定当前片段起始状态：人物站位、姿势、服装状态、场景空间和光源方向。尾帧用于锁定当前片段结束状态：人物最终姿态、情绪落点、道具状态和画面构图。中间过程必须自然连接首尾帧，不得提前到达尾帧，也不得偏离尾帧目标。
```

## 参考图污染失败类型

如果生成结果出错，按以下类型诊断：

```text
identity pollution：服装图或风格图污染人物脸。
costume pollution：人物图错误继承旧服装，未换成目标服装。
style override：风格图覆盖人物身份或服装结构。
pose override：动作参考图导致身体比例或脸部身份变化。
environment override：场景图改变人物身份或服装。
first-last conflict：首帧和尾帧状态无法自然连接。
mixed-average character：多参考图平均混合成一个新角色。
face integrity failure：参考图或生成结果出现人脸畸形、五官错位、脸部异常色块、黑斑、异常阴影、大面积死黑、皮肤塑料/蜡像/橡胶质感或肤色断层。
```

## 与人物镜头强制覆盖规则的关系

本规则负责“参考图各管什么”。
人物镜头强制覆盖规则负责“人物镜头必须写什么”。

如果参考图中有人物，并且最终画面也有人物，两个规则同时执行：

```text
先分配参考图职责
再执行人物镜头强制覆盖规则
再进入最终提示词结构
```
