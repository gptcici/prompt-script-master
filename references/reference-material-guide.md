# 参考资料建议库

复述阶段必须判断用户提供了哪些参考资料，并说明还建议补充哪些资料。

## 分级

- 必须资料：缺少会明显影响结果，例如固定人物图、产品定型图、首帧、尾帧、关键品牌视觉。
- 强建议资料：能显著提升稳定性，例如音乐片段、舞台灯光参考、动作参考、场景参考、材质参考。
- 可选资料：用于增强风格一致性，例如色彩参考、氛围参考、镜头参考。

## 按项目建议

- 舞台 / MV / 唱歌 / 跳舞：音乐或歌曲风格、歌词段落、节奏速度、歌手造型、舞台灯光、动作参考。
- 产品广告：产品图、材质细节、品牌视觉、目标受众、使用场景、logo / packshot 要求。
- 人物情绪：人物图、服装妆造、场景图、情绪参考、光线氛围。
- 剧情短片：角色设定、场景参考、故事段落、分镜或导演脚本。
- 首尾帧过渡：首帧、尾帧、主体一致性要求、过渡路径、禁止变化项。

## 表达规则

资料建议必须标注“我建议……”，不能让用户误以为建议内容来自原始设定。资料建议属于确认前沟通层，不能进入最终提示词。


## 本地素材库规则

当用户提供脚本拆解、参考图片、视频文件、关键帧或素材包时：

1. 先判断它们属于动作参考、构图参考、灯光参考、叙事结构参考、音乐节奏参考、人物一致性参考还是场景参考。
2. 读取并遵守 `reference-isolation-rules.md`。
3. 需要长期保留到技能包时，按 `asset-library-guide.md` 的目录和命名规则整理。
4. 大视频优先转为 proxy 视频、关键帧和 video-index，不直接塞入大型原文件。
5. 复述阶段可以说明“已识别为动作/镜头/光影参考”，但最终提示词中不得出现来源文件名或来源内容。
