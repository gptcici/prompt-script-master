# Camera Shot Contract 镜头合约

## 目的

本文件用于把“镜头感 / 电影感 / 运镜 / 分镜 / 景别”转成可执行的镜头选择。它不是镜头术语库，而是镜头决策规则：先判断画面任务，再决定景别、机位、焦点、运镜、起点、终点和风险。

## 启动条件

出现以下任一情况时读取：

- 用户要求电影感、镜头语言、景别、运镜、转场、分镜、镜头脚本、shot list、storyboard。
- 用户给出多镜头时间轴、首尾帧、关键帧、连续片段。
- 用户要求推镜、拉镜、横移、跟拍、环绕、航拍、俯拍、仰拍、特写、远景。
- 生成内容包含锁脸、口型、手部、产品细节、复杂动作、多人调度、强物理动力学或专业广告 / MV。

## 镜头合约字段

最终提示词不必逐项列标题，但内部必须先判断：

```text
【镜头合约】
shot_intention：这个镜头唯一要让观众感受到 / 看懂 / 发现 / 改变的是什么？
subject_priority：主体优先级，人物身份 / 表情 / 动作 / 产品 / 空间 / 材质 / 光影 / 动力学。
shot_size：景别，必须能承载主体优先级。
angle：机位角度，平视 / 低角度 / 高角度 / 俯视 / 仰视 / 侧面 / 背面 / 过肩等。
lens_feel：焦段感或空间压缩感，写成可见空间效果，不强制真实焦段。
camera_support：稳定机位 / 手持 / 轨道 / 摇臂 / 航拍 / 虚拟动画层移动。
movement：运镜方式和速度。
start_frame：镜头起始画面。
end_frame：镜头结束画面或动作落点。
focus_plan：焦点在谁/什么上，是否有焦点转移。
risk_control：该镜头需要保护什么，锁脸 / 口型 / 手部 / 文字 / 产品 / 动力学 / 轴线。
```

## 核心原则

1. 镜头为内容服务，不为术语服务。
2. 一个镜头只承担一个主任务：建立空间、推动动作、揭示信息、情绪靠近、展示材质、完成转场。
3. 景别必须匹配任务：远景不要求细微口型，特写不安排大幅走位。
4. 运镜必须有目的：推近表示靠近、发现、压迫或情绪进入；横移表示同行、穿越、空间关系；环绕表示主体雕塑感或产品英雄感。
5. 锁脸、口型、手部细节、文字和产品结构优先时，镜头应更稳定，避免过度旋转和抖动。
6. 强动作、风动、流体、粒子、多人调度同时出现时，镜头应减少一个自由度，优先保护主体清晰。
7. 镜头必须有起点和终点，不只写“镜头运动”。

## 正确写法

```text
0-3秒，中近景平视稳定推近，镜头从女主半身与古筝同框开始，焦点先落在她停在琴弦上的右手，再随她缓慢抬眼回到脸部。镜头目的不是展示环境，而是让观众感到她情绪从克制转向决心；推近速度很慢，终点停在她眼神光清晰的近景。
```

## 错误写法

```text
电影感镜头，推拉摇移，大片运镜，氛围很好。
```

问题：没有景别、机位、主体、起点、终点、焦点和镜头目的。
