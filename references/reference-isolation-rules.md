# 参考库隔离规则

本文件是最高优先级的参考库隔离规则，适用于所有 `references/` 中的脚本拆解、动作库、分镜库、灯光库和案例库，以及用户在当前对话中上传的参考资料。

## 最高规则

所有库内容只允许作为内部导演方法参考，不允许作为最终生成内容直接出现。

最终输出中不得出现：

- 参考库文件名、目录名、原始视频名、原始图片名。
- 参考素材中的真实艺人、演员、歌手、品牌、节目、MV、演唱会或站点名称。
- 可识别的原片桥段、原片时间戳、原片道具组合、原片歌词、字幕、logo、水印、作者名。
- “参考某某视频 / 按某某歌手 / 像某某MV / 按某某演唱会”这类来源痕迹。
- 原始拆解里的动作标签、库名、章节名、表格结构和训练样本文字。

## 允许使用的抽象信息

只允许把库内容转化为抽象导演原则：

- 情绪结构：引入、铺垫、转折、爆发、释放、收束。
- 镜头结构：大全景建立、中景承载、近景聚焦、特写放大、观众桥接、空间收束。
- 动作强度：轻动态、中动态、高动态。
- 表演逻辑：握麦、站位、缓步、抬头、闭眼、伸手、转身、与观众互动等通用动作。
- 光影逻辑：追光、逆光、侧光、顶光、彩色 wash、火光、日落光、自然光、烟雾光束等通用摄影语言。
- 剪辑逻辑：主歌慢、预副歌加速、副歌密集、间奏回到长镜头、结尾混剪释放。
- 空间逻辑：主空间作为情绪锚点，副空间作为记忆 / 幻想 / 对照，结尾用统一动作或节奏合并空间。

## 转译要求

当读取库内容后，必须先做一次内部转译：

1. 删除来源身份。
2. 删除具体原片场景和专属桥段。
3. 提取可泛化的导演方法。
4. 用用户当前项目的人物、故事、音乐、场景和情绪重新表达。
5. 最终输出只保留用户项目内成立的内容。

## 例子

错误：
“参考某歌手演唱会的国风段落，红色书法LED背景，粉色汉服持伞。”

正确：
“以东方叙事感处理这一段：镜头先用舞台远景建立仪式感，再切到人物侧脸中近景，动作保持缓慢克制，灯光以低饱和暖红和侧逆光制造戏剧张力。”

错误：
“像 Time Machine MV 那样让多个男人在浴缸、天桥、海边、卧室里轮流出现。”

正确：
“采用多空间幻想并置结构：主空间作为情绪锚点，多个副空间分别承载不同关系状态，每个副空间通过一个夸张但可拍摄的动作完成情绪对照，最后用统一节奏把空间收束。”
