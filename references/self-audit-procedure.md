# 技能自检审计规程

> 本文件定义 prompt-script-master 技能的定期自检流程，确保无旧规则残留、无跨文件冲突、所有引用链路完整闭合。

## 触发条件

- 任意规则文件有重大新增 / 改写 / 删除后（≥3 处修改）
- 用户主动要求"自检"或"检查是否有旧规则残留/冲突"
- 版本迁移（如 v0.9→v0.10 格式迁移）完成后
- 建议每 5-8 次编辑周期执行一次

---

## 审计三步法

### 第一步：旧规则残留扫描

用 `search_files` 在技能根目录搜索以下旧版特征模式：

| 搜索模式 | 含义 | 期望结果 |
|----------|------|---------|
| `9项格式` | 旧版时序格式 | 仅在废弃声明中出现 |
| `三段式` | 旧版输出结构 | 0 结果 |
| `全程总定调` | 旧版内容结构 | 仅在废弃声明中出现 |
| `全局收尾` | 旧版内容结构 | 仅在废弃声明中出现 |
| `主光方向.*轮廓光.*面部明暗` | 旧版光影描述格式 | 0 结果（应全部替换为锚点推导四段式） |
| `(主光方向:1\\.` | 旧版权重格式 | 0 结果 |

**判定**：任何旧版特征词出现在非 `已废弃` 声明上下文中 = 残留，必须修复。

### 第二步：跨文件冲突检查

| 检查项 | 方法 | 冲突迹象 |
|--------|------|---------|
| 权重上限 | 搜索 `1\.4`、`1\.8`、`≤ 1\.2` | 与新 S/A/B 体系 1.3 红线冲突 |
| 权重范围 | 搜索 `1\.1.*1\.15`、`1\.15.*1\.25` | 是否与 `seedance-prompt-order-rules.md` 定义一致 |
| 时间轴项数 | 搜索 `7 项` vs `9 项` | 凡出现 9 项格式 = 废弃标记是否到位 |
| 输出结构 | 搜索 `正文提示词.*时间轴.*负面` | 是否为两段式，非旧版三段式 |

**判定**：凡出现与权威规则文件不一致的数值/格式定义 = 冲突，以权威文件为准修正。

### 第三步：引用链路完整性

| 检查项 | 方法 |
|--------|------|
| 文件引用是否存在 | 对每个 `见 xxx.md`、`` `xxx.md` `` 类引用，用 `search_files` 确认对应文件存在 |
| Related 段完整性 | 检查每个规则文件的 `## Related` 段是否包含其上游规则引用 |
| 联动清单覆盖 | 对照 SKILL.md 维护注意事项中的联动清单，确认每个关联文件已登记 |
| 快照一致性 | `diff -rq` 对比主目录和快照目录对应路径 |

**判定**：引用不存在 = 必须修复（替换引用或创建缺失文件）；Related 段缺失上游引用 = 补充。

---

## 修复优先级

1. **拦停红线**（引用文件不存在、权重上限冲突）→ 必须先修复再继续
2. **旧版残留**（光影描述旧格式、未标记废弃的 9 项引用）→ 逐一替换为新版
3. **描述过时**（规则描述未覆盖新版增量的维度）→ 补充引用新规则文件
4. **关联不完整**（Related 段缺上游规则、脚本关键词未覆盖新版）→ 补充

---

## 审计后动作

1. 修复所有发现的问题
2. 同步快照（`skill/prompt-script-master/` 下对应路径），排除 `SKILL.md`
3. 更新记忆（`memory` 工具）：如有新的规则体系或文件关系，补充到 memory
