
# Retake Iteration Protocol 重拍迭代协议

## 目的

本规则用于生成结果出来之后的二次修复。它补充 `retake-failure-diagnosis.md`：先诊断失败类型，再决定是保留、后期修、局部编辑、换种子、改提示词，还是拆镜头。

## 启动条件

出现以下任一情况时读取本规则：

```text
重拍、retake、再来一版、这版部分可以、只修某个问题、换一版、继续调、复盘、失败、修脸、修动作、修光影、修风向、修连续性、已经生成但不满意
```

## 五类判定

每次生成结果先判定属于哪一类：

| 判定 | 适用情况 | 下一步 |
|---|---|---|
| 保留 | 核心目标达成，只有轻微小问题 | 锁定作为可用版本 |
| 后期修 | 问题属于颜色、裁切、字幕、声音、局部瑕疵 | 不重跑整条，交给后期或局部编辑 |
| 局部编辑 | 构图和动作正确，只有一个层面错 | 只改失败层，如脸、衣服、风、道具 |
| 换种子 / 重抽 | 提示词正确但抽样偶发失败 | 同提示词换一次或少量重抽 |
| 改写提示词 | 同类失败重复出现 | 回到失败诊断，改对应模块 |

## 单变量原则

每次重拍只改一个变量：

```text
只改提示词的一处关键模块，或只换 seed，或只换参考图职责，或只换镜头拆分方式。
```

不要同时改脸、动作、灯光、风、构图和参考图，否则无法判断是哪一处修复有效。

## 尝试预算

默认规则：

- 同一个失败点连续出现两次，就不要继续赌运气，应改提示词或拆镜头。
- 复杂 15 秒视频不适合无限重抽；优先用短片段验证动作、光影和脸。
- 脸、手、Logo、产品、对口型失败时，优先降低动作复杂度或锁定镜头。

## Take Log 记录格式

内部可记录：

```text
Take 1：改动 = 原始版本；结果 = 脸像但风乱；判定 = 改写风动模块。
Take 2：改动 = 只强化风向；结果 = 风统一但口型弱；判定 = 局部修口型。
```

用户不要求时，不展示完整日志，只输出归因和新版提示词。

## 和失败诊断的关系

执行顺序：

```text
retake-failure-diagnosis：判断错在哪里。
retake-iteration-protocol：决定怎么修、修多少、是否重跑。
```

## 输出格式

用户要求解释时：

```text
【本次问题】
...

【判定】
保留 / 后期修 / 局部编辑 / 换种子 / 改写提示词

【只改一个变量】
...

【新版提示词】
...
```

用户只要新版提示词时，只输出最终可复制提示词。
