# Prompt Budget Allocation 提示词预算分配

## 目的

提示词不是越长越好。每条生成都有有限的注意力和保真预算。本文件用于决定当前镜头或当前 clip 的预算应该优先花在身份、动作、场景、光影、动力学、参考图还是声音上。

## 启动条件

出现以下任一情况时读取本文件：

```text
复杂画面 / 多参考图 / 多人物 / 多镜头 / 15秒以上 / 提示词太长 / 信息太多 / 同时要锁脸和大动作 / 同时要多人和细节 / 审查提示词预算 / 生成泛化
```

## 三大预算消耗

| 预算方向 | 买到什么 | 牺牲什么 |
|---|---|---|
| 身份保真 | 稳定脸、产品、服装、logo、妆容 | 动作幅度、多人、复杂背景 |
| 动作强度 | 大幅动作、舞蹈、武打、运动轨迹、物理结果 | 近景脸部细节、手部稳定、服装一致性 |
| 场景密度 | 群像、街市、天气、道具、层次丰富的世界 | 单个主体精度、可读文字、小物件稳定 |

## 预算分配方法

每个镜头或当前 clip 必须先内部判断：

```text
primary spend：当前镜头最重要的一个目标
secondary spend：当前镜头可以兼顾的一个次目标
economized：当前镜头故意压缩或不追求的内容
reference offload：哪些信息由参考图承担，文字不再重复花预算
split decision：是否必须拆成两个镜头或两个 clip
```

## 默认分配策略

### 1. 锁脸 / 角色定妆优先

当用户提供人物参考图、三视图、定妆图、要求“脸必须像”时：

```text
primary spend：人物身份保真
secondary spend：表情 / 服装 / 光影中的一个
economized：大幅动作、复杂群像、过多背景细节
```

写法：人物脸和发型交给参考图，文字只补充当前动作、表情变化、光源和镜头。

### 2. 动作 / 舞蹈 / 武打优先

```text
primary spend：动作轨迹和物理结果
secondary spend：身份参考锚点
economized：近景面部微表情、复杂饰品细节、多人同时大动作
```

写法：动作要有起点、力量、路径、结果和终点；脸部用中景或短切近景补偿。

### 3. 场景 / 世界观优先

```text
primary spend：空间、天气、建筑、气氛
secondary spend：镜头运动或光影
economized：远景人物脸、手部细节、小字、小 logo
```

写法：人物只作为轮廓或队形出现；重要人物细节另开镜头。

### 4. 对话 / 唱歌 / 对口型优先

```text
primary spend：脸部稳定 + 口型 / 歌唱节奏
secondary spend：情绪表演或面部光影
economized：复杂运镜、头部大幅转动、多人同时说话、手部复杂动作
```

写法：锁定中近景或近景，短句，少头动，嘴唇开合、下颌稳定、呼吸节奏清楚。

### 5. 产品 / 服装 / 道具展示优先

```text
primary spend：产品 / 服装 / 道具结构和材质
secondary spend：一个动作或一个光影变化
economized：大幅剧情、群像、复杂背景
```

写法：单独给材质、接触点、反光和结构镜头；不要同时要求大剧情。

## 拆分触发

必须建议拆分的情况：

- 一个 5-8 秒镜头同时要求锁脸、跳舞、换装、落泪、转场、复杂场景。
- 一个镜头内有多次完成动作：走进、坐下、弹琴、落泪、起身、离开。
- 多人同时大动作，还要每个人脸清楚。
- 产品证明、情绪剧情、英雄 packshot 同时发生。
- 首帧到尾帧之间缺少自然物理桥接。

## 最终提示词预算原则

- 先写必须控制的内容，后写氛围。
- 参考图已经能表达的静态信息不要大量重复。
- 镜头内只保留一个主要事件。
- 远景不要求脸部极致保真；特写不要求复杂场景密度。
- 把未来故事、背景设定、创作意图留在复述或规划，不塞进当前最终提示词。
