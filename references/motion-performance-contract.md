
# Motion Performance Contract 动作表演合约

## 目的

本规则用于人物动作、道具动作、舞蹈、武打、手部动作、产品演示、环境动态和物理运动。它把“动起来”改写成可被视频模型执行的物理动作：谁动、怎么动、力量多大、何时发生、造成什么结果、最后停在哪里。

## 启动条件

出现以下任一情况时读取本规则：

```text
人物动作、手部动作、走位、舞蹈、打斗、武器、转身、回头、抬眼、奔跑、坠落、拿起、放下、推开、拉开、飞起、产品演示、道具运动、物体碰撞、雨雪烟尘、水面、动作参考视频、动作不连贯、动作重拍
```

## 动作合约字段

任何关键动作都应内部明确以下字段。最终提示词不一定逐项列出，但必须在句子中体现。

```text
动作主体：谁或什么在动。
动作动词：具体做什么。
力量级别：轻 / 中 / 强，或克制 / 急促 / 爆发。
时间窗口：在几秒内发生，或在音乐哪个节拍发生。
物理结果：动作造成什么可见结果。
连续要求：是否接上上一镜头 / 上一段动作。
动作终点：最后停在哪里，形成什么画面状态。
```

## 强动作写法

强动作 = 动作 + 物理结果 + 终点。

弱写法：

```text
她很紧张地站着。
```

强写法：

```text
她吸气后指尖收紧衣袖，肩膀轻微下沉，眼神避开镜头半秒后重新抬起，最后停在克制凝视的状态。
```

弱写法：

```text
纸张被风吹动。
```

强写法：

```text
桌角纸张被右前方来风掀起，边缘先抬起再翻折一次，最后滑到杯子旁停住。
```

## 物理优先原则

先写物理原因，再写可见结果。动作必须让画面发生可见变化。

示例：

```text
厚重木门被风推开，门轴缓慢转动，门缝气流压弯烛火，烟线向走廊方向卷去。
```

比以下写法更强：

```text
门打开，很有戏剧感。
```

## 三段动作节奏

短视频动作优先使用三段节奏：

```text
起始状态 → 动作发生 → 改变后的结束状态
```

示例：

```text
0-2秒，琴弦静止，女主手指悬在弦上。
2-4秒，右手指尖轻拨琴弦，袖口被风掀起，琴弦震动形成细小高光。
4-6秒，琴弦余震渐弱，女主手指停在弦边，眼神缓慢抬起。
```

## 人物动作规则

人物动作必须服务角色目标和镜头意图。

优先写：

- 头、眼、眉、唇、肩、手、身体重心的联动。
- 一个清楚的动作落点。
- 动作背后的潜台词。
- 动作和表情的时间差。

避免：

- 多个大动作挤在同一短镜头。
- 表情、手、身体、镜头同时剧烈变化。
- 唱歌/说话时大幅转头导致口型和脸崩。
- 手部复杂交互、快速手指动作、多人递物。

## 手脸稳定策略

当镜头重点是脸、口型、锁脸、人物身份或产品 Logo 时，降低动作复杂度。

- 口型 / 肖像镜头：镜头稳定，头部转动小，手部只做简单动作。
- 锁脸换装镜头：动作幅度中低，避免遮脸、摸脸和大幅旋转。
- 产品镜头：产品主体保持几何稳定，可移动灯光、反射或环境，而不是移动产品本体。
- 手部特写：只做一个动作，如拨弦、握住、放下、拂过，不叠加多动作。

## 道具和环境动作

道具动作必须有状态变化：

```text
杯子从桌面被拿起 → 水面轻晃 → 杯底离开桌面留下圆形水印。
```

环境动作必须有物理来源：

```text
风从右前方吹来 → 纱幔先鼓起 → 远处云雾同向缓慢流动。
```

## 动力学扩展

当动作造成可见物理结果时，必须联动 `physical-dynamics-standard.md`：

- 手触碰道具：写接触点、受力方向、反作用、停止位置。
- 脚踩水面：写落点、涟漪扩散、反射破碎与回合。
- 推门 / 拉帘：写转轴、摩擦、气流、光束、烟线或布料响应。
- 拨弦 / 挥袖 / 拔剑：写震动、惯性、延迟、回弹、最终静止状态。
- 雨雪烟火粒子：写来源、路径、密度、受光、落点或消散。

动作合约负责“谁做什么”；物理动力学标准负责“动作造成什么可见物理后果”。

## 与连续片段的关系

若触发 `sequence-state-capsule.md`，动作合约必须继承上一段的动作状态：

- 不重复已经完成的动作。
- 不提前执行保留给未来的动作。
- 上一段未完成的动作向量可延续到当前段。
- 当前段必须有新的动作终点。

## 负面提示词建议

```text
动作类：动作无落点、动作重复、动作卡顿、肢体和表情脱节、手指崩坏、快速复杂手部动作、身体重心漂浮、动作没有物理结果、上一段动作重演
```
