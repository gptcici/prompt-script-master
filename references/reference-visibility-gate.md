# Reference Visibility Gate 参考图可识别性门（V0.9.82）

## 目的

本规则用于所有图生视频、首尾帧、参考图锁脸、换装、多参考图融合、图片重绘、角色图、三视图和关键帧任务。它解决两个核心风险：**图片信息不清晰时，不能盲目执行 I2V 极简原则；图片大面积不清晰时，应建议用户重新生成或上传更清晰的参考图**。

I2V 极简原则只有在参考图中的关键视觉信息清晰可识别时才成立。如果图片里的脸、服装、发型、背景、构图或动作起点不清楚，最终提示词必须补写不清晰部分，不能只写“保持图中内容”。

## 启动条件

只要满足以下任一条件，就先启动本规则：

```text
用户上传图片 / 首帧 / 尾帧 / 人物图 / 服装图 / 发饰图 / 场景图 / 构图图
用户要求图生视频、动起来、续写、首尾帧过渡、锁脸、换装、多图融合、角色连续、图片重绘、三视图或关键帧
用户说“参考图”“图1/图2”“保持图片里的人物/服装/背景/构图”
```

## 可识别性检查维度

### 1. 人脸可识别性

检查脸部是否清晰可辨：

- 脸部是否足够大，五官是否可见。
- 正脸、侧脸或三分之二侧脸角度是否可判断。
- 眼睛、鼻梁、嘴唇、脸型边界是否被遮挡、模糊、过曝或死黑覆盖。
- 参考图是否存在畸形脸、错位五官、不符合生理逻辑的面部结构。
- 皮肤上是否有不协调色块、黑斑、异常阴影、大面积黑色、脏污式色块、塑料质感或蜡像质感。

如果人脸不可识别或存在明显异常，不能只写“保持图中人物身份”。必须补写人物身份锚点，并触发 `face-integrity-check.md`。

### 2. 服装可识别性

检查服装是否可辨：

- 领口、肩线、袖型、腰部、裙摆或裤型是否完整可见。
- 材质、颜色、纹样、层次、装饰、披帛或外袍是否清楚。
- 是否被裁切、遮挡、低清晰度或强阴影影响。

服装不清晰时，不能只写“保持图中服装”，必须补充服装结构、材质、颜色和关键轮廓。

### 3. 发型 / 发饰可识别性

检查发型、发量、发长、发色、发髻、刘海、发带、发簪、流苏、头饰位置是否清楚。若不清晰，提示词必须补写发型和发饰关键锚点。

### 4. 背景 / 空间可识别性

检查场景类型、前中后景、地面、墙面、门窗、家具、山水、建筑、光源位置和空间透视是否清楚。若不清楚，必须补写场景类型、空间层次、光源方向和主要环境锚点。

### 5. 构图 / 景别可识别性

检查主体位置、画幅比例、景别、镜头角度、裁切边界、运动空间、前景遮挡和视觉重心是否明确。若不清楚，必须补写构图关系和主体位置。

### 6. 首尾帧连接性

首尾帧任务必须检查：主体是否一致、服装和场景是否能衔接、首帧动作起点和尾帧动作终点是否存在可行路径、光源方向和构图是否冲突。若冲突，必须先提示风险或改成分段过渡。

## 四档决策

### A. 清晰可识别

```text
参考图信息清晰：执行 I2V 极简原则。
只写图片看不出来的动作、运镜、光线变化、动力学和保持约束。
```

### B. 部分可识别

```text
清晰部分不重复写；不清晰部分必须补写。
例如：脸清楚但服装不清楚 → 保持图中人物身份，同时补写服装轮廓、材质和颜色。
```

### C. 不可识别 / 存在异常

```text
不能执行 I2V 极简原则。
必须重建关键视觉锚点，或在确认阶段指出参考图不足。
如果是快捷生成，也必须在正文里补足身份、服装、发型、场景、构图或面部完整性锚点。
```

### D. 不清晰区域较多 / 不适合承担参考职责

```text
如果图片里人脸、服装、发型、背景、构图等关键区域有三项及以上不清晰，或用户指定的主职责区域不可辨，必须建议用户重新生成或上传更清晰的参考图。
不能把严重不清晰的图片继续当作锁脸图、服装图、场景图、首尾帧或构图强参考。
```

建议重生参考图时，应说明具体原因和改进要求：例如更清晰的脸部、完整服装、无遮挡主体、明确光源、非死黑背景、非过曝画面、首尾帧主体一致。

## 最终提示词写法

清晰参考图写法：

```text
图1作为首帧，严格保持图中人物身份、服装、发型、空间构图和光源方向；只让她右手缓慢抬起并回头，镜头轻微推近，纱幔和发带沿同一风向轻动。
```

部分不清晰写法：

```text
图1用于锁定人物脸型比例和眼神气质；由于服装下半身细节不可辨，正文补足为浅青色交领上衣、宽袖、层叠长裙和轻薄披帛，整体保持古风清雅质感。
```

异常参考图写法：

```text
参考图仅用于大致人物方向，不直接继承图中异常面部阴影和不自然肤色。生成时重建自然生理结构：五官对称、眼鼻唇位置合理、皮肤色调连续、面部阴影服从单一光源。
```

## 禁止行为

- 图片模糊仍只写“保持图中人物”。
- 脸部被遮挡仍只写“锁脸”。
- 服装被裁切仍只写“保持图中服装”。
- 背景死黑仍只写“保持背景”。
- 首尾帧主体不一致仍强行写自然过渡。
- 人脸存在黑斑、异常阴影或畸形结构时仍把这些当作身份锚点继承。


## 建议重新生成参考图的条件

当以下任一条件成立时，应在确认阶段建议用户重新生成或上传更清晰的参考图：

- 关键主体不清楚，且任务需要锁脸、换装、三视图、角色连续或首尾帧精确过渡。
- 人脸、服装、发型/发饰、背景/空间、构图/景别五类中有三类及以上不清晰。
- 用户指定某张图承担主职责，但该职责区域不可辨，例如锁脸图看不清五官、服装图看不清服装结构、场景图大面积死黑。
- 首尾帧主体不一致、动作路径不可连接，或两帧光源/构图冲突严重。
- 参考图存在大面积 AI 伪影、异常黑斑、脏污色块、压缩噪点、脸部死黑或不符合生理逻辑的皮肤质感。

如果用户明确要求继续使用低清图，必须降低该图职责权重，并在最终提示词中补写不清晰部分、排除异常区域。
