# Event Density Control 事件密度控制

## 目的

本文件用于判断一个镜头、一个 clip 或一条最终提示词到底能承载多少剧情和动作。它防止把太多事件塞进一个短视频，导致动作跳跃、身份漂移、未来事件提前出现或镜头失控。

## 启动条件

出现以下任一情况时读取本文件：

```text
多动作 / 多事件 / 复杂剧情 / 多地点 / 多段对话 / 5秒 / 8秒 / 10秒 / 15秒 / 连续剧情 / 多段 MV / 用户要求很多事情一次发生 / 一镜到底但动作很多 / 首尾帧距离太大
```

## 核心密度规则

一个 generation 通常只承载：

```text
一个可见核心事件 + 一个改变后的终点
```

长背景、动机解释、多地点旅程、未来结局、复杂对白、多个主要动作，应该进入 sequence plan，而不是当前最终提示词。

## 事件桶

连续或复杂项目必须内部区分：

```text
already_happened：已经发生，不得重演。
this_clip_only：当前 clip 可以执行的事件。
reserved_for_later：未来片段执行，不得提前出现。
do_not_show_yet：即使能解释动机，当前也不能展示。
```

## 时长建议

### 3-5 秒

适合：一个微动作、一个表情变化、一个道具接触、一个小转场。

不适合：完整剧情、复杂走位、多人互动、强对口型加大动作。

### 6-8 秒

适合：一个清晰动作链：起点 → 动作 → 结果。

例如：抬手触弦 → 琴弦震动 → 抬眼停住。

### 10-15 秒

适合：3-5 个镜头，1 个主要情绪推进或动作推进。

不适合：多个地点、多个剧情反转、完整起承转合。

### 15 秒以上

必须明确镜头切点、事件分配和片段终点。复杂剧情建议拆成多个 clip。

## 拆分触发器

遇到以下情况，必须建议拆分或在内部简化：

- 一个句子里出现 3 个以上完成动作。
- 既要走位，又要唱歌，又要跳舞，又要情绪特写。
- 多人轮流说话超过两轮。
- 首帧和尾帧差距需要多个中间动作才能自然到达。
- 一个镜头同时要求产品展示、剧情推进、情绪爆发和大转场。
- 场景从室内到室外再到另一个地点。
- 当前 clip 同时写了已发生事件和未来结局。

## 当前 clip 防火墙

故事背景可以影响：

- 表演动机
- 情绪基调
- 镜头压力
- 光影选择
- 声音层次

但不得让当前 clip：

- 表演未来事件
- 提前揭示未来信息
- 跳过必要物理交接
- 重演上一段动作
- 一次解决多个后续问题

## 正确示例

复杂需求：

```text
女主走进亭子，坐下弹琴，回忆过去，落泪，起身离开，远景消失在雾里。
```

拆分：

```text
Clip 1：女主走进亭子，停在古筝前。
Clip 2：女主坐下弹琴，情绪开始低落。
Clip 3：女主落泪后停下琴声。
Clip 4：女主起身离开，远景消失在雾里。
```

当前提示词只写当前 clip，不把四个 clip 全塞进一个 8 秒视频。
