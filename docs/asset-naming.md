# 素材命名与资产管理规则

## 核心原则

用户可能上传多张参考图、分镜图、首帧、尾帧、人物模板、产品图、场景图、风格图、音频或视频参考。

系统必须在拆解阶段主动识别素材类型，并为素材建立清楚的用途与优先级。

## 默认命名规则

建议采用以下命名：

```text
character_ref_01：人物模板图
product_ref_01：产品定型图
first_frame_01：首帧图
last_frame_01：尾帧图
storyboard_01：分镜图
scene_ref_01：场景参考图
style_ref_01：风格参考图
lighting_ref_01：灯光参考图
object_ref_01：物件参考图
audio_ref_01：音频 / 音乐节奏参考
video_ref_01：视频运动参考
```

如果同类型素材超过一张，按顺序编号：

```text
character_ref_01
character_ref_02
scene_ref_01
scene_ref_02
```

## 素材分类规则

系统收到素材后，应判断它属于：

1. 必须锁定参考
2. 强参考
3. 仅参考范围

并向用户说明：

> 我会把这张图作为人物模板图，属于必须锁定参考，主要锁定人物身份、脸部特征、发型、服装和核心气质。

或：

> 我会把这张图作为场景参考图，属于强参考，主要参考空间结构、光线方向和舞台布局，不锁定图中人物身份。

## 必须锁定参考

以下素材默认属于必须锁定参考：

- 人物模板图
- 角色定妆图
- 产品定型图
- 首帧图
- 尾帧图
- 品牌主视觉定型图

## 强参考

以下素材默认属于强参考：

- 分镜图
- 场景结构图
- 舞台结构图
- 镜头路径图
- 视频运动参考
- 音乐节奏参考
- 灯光氛围参考

## 仅参考范围

以下素材默认属于仅参考范围：

- 材质图
- 表情参考图
- 局部道具图
- 色彩参考图
- 光效参考图
- 氛围图

## 素材冲突处理

如果素材之间存在冲突，系统不能自动决定。

示例：

- 人物模板图是短发，但尾帧图是长发。
- 产品定型图是圆瓶，但广告参考图是方瓶。
- 首帧是室内影棚，但场景图是户外城市。
- 分镜图要求镜头从左向右，首帧人物却朝正前方静止。

系统必须指出冲突并询问用户保留哪个方向。

## 最终标准

每个素材都要有用途、优先级和锁定程度。  
不要把所有素材混成“参考这些图”。
