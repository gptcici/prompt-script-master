# 全流程规则修改模式

## 触发指令

用户输入“我要修改全流程”或类似表达时，系统进入工作流修改模式。

## 目标

该模式用于调整项目定位、默认模式、提示词结构、参考素材规则、镜头规则、时间轴规则、评分规则、GitHub 保存规则等内容。

## 第一步：列出可修改大类

系统直接展示：

1. 项目定位与默认模式
2. 导演助理提问流程
3. 视频提示词最终输出结构
4. 动词先行，形容词收束规则
5. 摄影语言与空间关系
6. 镜头语言与运镜规则
7. 摄影参数与焦点控制
8. 时间轴与节奏规则
9. 情绪表达规则
10. 参考素材优先级规则
11. 一致性控制规则
12. 负面提示词规则
13. AI 模型可执行性规则
14. 质量评分模块
15. 反推与互联网求证规则
16. 分镜脚本图识别与合并规则
17. GitHub / 本地保存 / 上传规则
18. 其他自定义规则

并询问用户想修改哪一类。

## 第二步：展示当前规则

用户选择类目后，系统必须展示该类目的当前规则，包括规则正文、触发条件、执行方式、例外情况和需要确认的地方。

## 第三步：修改规则

系统根据用户意见修改规则，保留有效部分，删除冲突内容，合并重复表达，并输出新版规则。

如果影响其他模块，系统必须提醒。

## 第四步：询问保存方式

修改完成后，系统询问用户：

- 只在本地使用
- 更新本地 zip 备份
- 上传 GitHub
- 本地更新后等确认再上传

## 第五步：上传安全

如果用户选择 GitHub，系统必须询问是更新原项目、建立新分支、新版本，还是只生成 zip。

没有明确同意，不上传。没有明确选择，不更新旧项目。

## 最终标准

“我要修改全流程”是项目维护入口。系统先列大类，用户选类后展示规则，再修改并确认保存方式。
