# Golden Tests

## 目标

Golden tests 用于真实上传 Skill 后验证输出是否符合预期。

这些测试不要求逐字一致，但要求输出结构、工作流判断和质量控制行为稳定。

## 全局硬性通过标准

无论哪一条测试，只要生成最终视频提示词，就必须满足：

- 时间轴内每个时间段都包含衔接关系、画面动作、场景动态、光线变化、镜头控制、摄影信息、情绪节奏。
- 摄影信息必须包含摄影机视角、焦段或镜头类型、景别变化、构图、焦点对象和景深状态。
- 不能只在单独的“镜头语言与摄影参数”段落里写运镜、焦段或焦点。
- 不能只在单独的“灯光、场景与动态”段落里写光线和场景。
- “镜头目标、整体风格、情绪基调”应合并成整体目标与风格基调。
- 如果时间轴没有告诉模型每一段如何衔接、怎么拍、光线怎么变、焦点在哪里，该测试视为不通过。

## 测试 1：演唱会英雄镜头

### 输入

```text
帮我把这个想法写成 Seedance 2.0 全能参考视频提示词：一个女歌手站在演唱会舞台中央唱副歌，背后有巨大的时间环，灯光很震撼。
```

### 预期行为

- 可以直接生成，不必须追问。
- 默认使用 Seedance 2.0 全能参考。
- 输出应包含生成规格、整体目标与风格基调、时间轴、一致性要求和禁止项。
- 应把“震撼”转化为可见画面：光线、舞台、观众灯海、时间环、镜头推进。
- 时间轴每一段都必须写明衔接关系、场景变化、光线变化、摄影机、焦段、运镜、景别、焦点和情绪推进。

### 通过标准

输出不是抽象文案，而是可复制的视频提示词；所有具体执行信息已经嵌入时间轴。

## 测试 2：首尾帧图生视频

### 输入

```text
我有首帧和尾帧，想让人物从坐着自然走到窗边，请帮我写图生视频提示词。
```

### 预期行为

- 不应直接硬写最终提示词。
- 应先询问或确认首帧、尾帧、人物身份、空间关系和动作路径。
- 应提醒首尾帧属于必须锁定参考。

### 通过标准

系统先确认关键素材关系，而不是直接生成。

## 测试 3：错误提示词优化

### 输入

```text
帮我检查这个提示词哪里不够好：高级电影感，震撼，燃，很好看。
```

### 预期行为

- 应指出问题：主体不明确、场景缺失、动作缺失、时间轴缺失、时间轴内镜头控制缺失、时间轴内光线和摄影信息缺失。
- 应给出优化方向。
- 如果生成改写版，应补齐主体、场景、动作、时间轴内光线、运镜、焦段、景别变化、焦点控制和禁止项。

### 通过标准

系统能识别抽象提示词不可执行，并转换为结构化提示词；不能只补单独的镜头语言段落或光线场景段落。

## 测试 4：产品广告镜头

### 输入

```text
帮我写一个 8 秒的产品广告视频提示词，主体是一瓶黑色香水，背景是深色大理石台面，感觉要高级、冷感、有奢侈品广告质感。
```

### 预期行为

- 可直接生成。
- 应把“高级、冷感、奢侈品广告质感”转化为材质、光线、反光、镜头推进、背景虚化。
- 时间轴应逐段写明产品如何被拍摄：镜头如何推进、用什么焦段、景别如何变化、焦点锁定瓶身、瓶盖、反光或品牌区域，台面反光和背景明暗如何变化。
- 禁止项应包含产品变形、品牌标识错位、材质错误、反光异常等。

### 通过标准

输出能保护产品形态和材质一致性，并且所有具体执行信息已经嵌入时间轴。

## 测试 5：多镜头需求

### 输入

```text
我要做一个 20 秒短片，开头是城市天台，中间人物奔跑，最后停在霓虹灯前回头看镜头。
```

### 预期行为

- 应识别 20 秒更适合拆成多镜头或分镜组。
- 应询问是否拆分，或按专业默认拆成 3 个分镜。
- 输出应包含镜头之间的连续性要求。
- 如果直接给分镜草案，每个分镜的时间轴都必须包含衔接关系、场景动态、光线变化、运镜、焦段、景别、焦点和情绪节奏。

### 通过标准

系统不应强行写成一个过长单镜头；如果给出分镜，每个分镜必须有完整时间轴。
