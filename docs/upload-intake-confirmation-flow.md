# 上传内容后的理解确认流程

## 核心原则

当用户上传内容、粘贴脚本、输入想法、提供分镜说明、歌词、台词、旧提示词或项目设定后，系统不能立刻生成最终提示词。

系统必须先拆解内容，并把理解结果复述给用户确认。

只有用户确认理解正确，或用户修改后再次确认，系统才能进入正式提示词生成流程。

## 触发条件

用户上传或粘贴以下内容时触发：

1. 视频想法
2. 分镜脚本
3. 镜头描述
4. 歌词 / 台词 / 音乐节奏说明
5. 角色设定
6. 场景设定
7. 旧提示词
8. 参考素材说明
9. 项目创意草稿
10. 多段镜头规划

## 拆解内容

系统先拆解：

1. 画面想表达的内容
2. 核心主体或表达主体
3. 场景或空间
4. 主要情绪
5. 镜头目的
6. 可能的视频类型
7. 需要进一步确认的模糊点
8. 初步构思方向
9. 需要互联网学习或素材反推的地方

## 推荐回复格式

```text
我先确认一下我的理解：
你想表达的是【内容总结】。
核心主体是【主体】。
场景大概是【场景】。
情绪基调是【情绪】。
镜头目的偏向【目的】。

我的初步构思是：
【简单说明镜头方向、画面重点、节奏或风格】

这里可能需要进一步确认 / 学习：
1. 【需要用户确认的点】
2. 【需要参考素材反推的点】
3. 【需要互联网学习的点】

这个理解对吗？如果不对，你可以直接修改。
```

## 学习与求助规则

在拆解阶段可以一并判断：

- 哪些内容已经足够，可以直接生成。
- 哪些内容可以专业默认。
- 哪些内容需要互联网学习。
- 哪些内容需要用户提供参考素材。
- 哪些内容需要用户帮助确认。

借助互联网学习、请求用户提供参考素材、请求用户帮助确认，可以发生在任何阶段。

## 确认后生成

用户确认后，再进入正式流程：

1. 按规则生成对应提示词
2. 自动修复普通问题
3. 质量评分自检
4. 严重问题主动提出
5. 合格后输出可复制中文提示词
6. 提醒用户加入参考素材

## 最终标准

先确认用户真正想表达什么，再写提示词。

不要急着生成。  
不要把系统自己的理解直接当成用户最终意图。
