# 反推与不确定性处理规则

## 核心原则

当系统无法可靠判断镜头、风格、节奏、空间关系或参考素材用途时，不应直接硬写最终提示词。

应先寻找依据，再生成。

## 需要补充依据的情况

以下情况需要先补充依据或请用户确认：

1. 用户指定某种导演、电影、MV、广告或视觉风格，但没有提供足够说明。
2. 用户上传的分镜图或运动路径不清楚。
3. 首帧和尾帧之间的空间、人物或动作关系不明确。
4. 产品、人物、场景或品牌视觉必须保持高度一致。
5. 镜头设计存在多种合理方向，且会影响最终结果。
6. 如果继续生成，只能堆抽象形容词，无法形成可见画面。

## 处理方式

系统可以使用以下方式补充依据：

- 请用户提供人物模板图、产品图、首帧、尾帧、分镜图或场景图。
- 请用户提供电影截图、MV截图、广告案例或风格参考图。
- 在可用时查找公开资料或真实案例，并把案例中的镜头、光线、空间和节奏转化为提示词语言。
- 如果用户明确要求系统决定，则只对非核心专业参数使用默认值。

## 不确定性说明

如果某项内容无法确认，系统应说明不确定点，并给用户 2-3 个可选方向。

示例：

```text
这里有一个不确定点：你想要的是稳定推进的广告镜头，还是更强烈的环绕英雄镜头？
A. 稳定推进
B. 环绕英雄镜头
C. 你帮我按专业方式决定
```

## 最终标准

不确定时先说明，不硬编。  
能默认的专业参数可以默认。  
会改变核心画面意图的内容必须确认。
