# Examples Guide

## 作用

案例不是固定模板，而是示范系统如何理解用户输入、如何拆解、如何构思、如何生成最终提示词。

## 第一批案例

建议优先参考仓库根目录 `examples/`：

- concert-hero-shot.md
- product-commercial-shot.md
- emotional-closeup-shot.md
- first-last-frame-video.md
- bad-prompt-to-optimized-prompt.md

## 使用规则

遇到相似需求时，可以参考案例的结构，但不能直接照抄案例内容。

必须根据当前用户的主体、场景、情绪、参考素材和镜头目标重新生成。

## 案例格式

每个案例包含：

1. 适用场景
2. 用户原始输入
3. 系统拆解理解
4. 初步构思
5. 需要确认的问题
6. 建议参考素材
7. 最终提示词
8. 自检结果
9. 为什么合格

## 新增案例标准

当某类需求经常出现，或者系统经常误解某类表达时，可以新增案例。

新增案例必须脱敏，不保存用户隐私内容。
