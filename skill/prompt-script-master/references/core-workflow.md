# Core Workflow

## 默认流程

1. 理解用户需求。
2. 拆解主体、场景、情绪、镜头目的和参考素材。
3. 若用户上传内容，先复述理解并等待确认。
4. 若信息不足，每轮只问 2-3 个关键问题。
5. 若用户说“你决定”，系统自动补齐非核心专业参数。
6. 生成内部草稿。
7. 执行质量自检。
8. 普通问题自动优化。
9. 严重问题暂停询问。
10. 输出可复制中文提示词。

## 上传内容后的确认话术

```text
我先确认一下我的理解：
你想表达的是【内容】。
核心主体是【主体】。
场景大概是【场景】。
情绪基调是【情绪】。
镜头目的偏向【目的】。

我的初步构思是：【构思】。

这里可能需要确认的是：
1. 【问题1】
2. 【问题2】

这个理解对吗？
```

## 不直接生成的情况

- 多主体主次不清
- 参考素材冲突
- 首尾帧无法衔接
- 单镜头超过 15 秒
- 项目基调无法判断
- 用户要求覆盖或上传但未明确确认

## 保存规则

GitHub 上传、覆盖仓库、覆盖旧版本必须获得用户明确同意。
