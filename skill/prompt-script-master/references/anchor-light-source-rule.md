# 锚点光源规则（Anchor Light Source）

## 核心原则

**一条提示词，一个锚点光源。所有光影、阴影、面光、环境光、轮廓光，必须从该锚点光源推导而来，禁止出现第二个不相关的主光源。**

> 本规则不给任何现有规则"加逻辑"——只要求所有光影相关规则在生成内容前，先对齐本规则定义的唯一锚点光源。

---

## 一、锚点光源的定义

锚点光源 = 提示词中最早明确出现的、具备完整三要素的主光源描述。

| 三要素 | 说明 | 示例 |
|--------|------|------|
| **类型** | 光源种类 | 夕阳、天光、月光、灯笼光、烛光、窗光 |
| **方向** | 光从哪个方向来 | 侧逆光、顶光、侧方光、正面光 |
| **位置** | 光从哪个具体位置发出 | "从亭柱缝隙切入""从悬崖外天空洒落""从左侧窗格投射" |

### 锚点光源的标准写法

```text
[光源类型]从[具体位置][方向]：[效果描述]
```

> ⚠️ 以下示例仅参考格式，实际根据场景独立撰写。

**有效锚点示例：**

```
夕阳侧逆光从亭柱缝隙切入，在人物身后拖出细长暖色阴影
灯笼暖黄光从左侧廊下悬挂处洒落，照亮半张侧脸
阴天柔和天光从云层后方均匀弥散，无明显方向性硬光
```

**锚点无效示例（不可接受）：**

```
暖金色光线笼罩人物              ← 无类型、无方向、无位置
人物面部立体光影自然              ← 只有效果，无光源
侧逆光打亮轮廓                    ← 有方向有类型，缺位置锚点
```

---

## 二、锚点光源的建立时机

锚点光源必须在提示词的**开场段落**（视频规格和场景总设定部分）中确立，不得晚于第一个时间轴段落。

**提示词结构中的强制位置：**

```text
视频规格 + 全程运镜总基调
@Image1 参考图指令
人物核心外形特征
整体场景总设定              ← 锚点光源必须在此区域内确立！
───
时间轴段落 1
……
```

锚点确立后，时间轴段落内不得重新定义光源，只能引用和细化。

---

## 三、锚点光源的统领规则

锚点光源确立后，其他所有光影相关内容的生成逻辑如下：

### 3.1 人物面光 / 骨骼光影

> 参照规则：`seedance-closeup-face-lighting-rules.md`

生成前先验证：
- 面光的方向是否与锚点方向一致？
- 面光的冷暖是否与锚点类型匹配？
- 面光的强度是否与锚点场景条件匹配？

**正面示例：** 锚点为"夕阳侧逆光从亭柱缝隙切入" → 面光写"侧后方暖金光线勾勒鼻梁和颧骨高光，正脸处于柔和半暗"
**反面示例：** 锚点为"夕阳侧逆光" → 面光写"正面柔光铺脸，五官均匀明亮"（方向矛盾）

### 3.2 环境光影

环境光影的方向、色温、阴影投射方向，必须与锚点光源保持一致。

- 锚点是"夕阳侧逆光从悬崖外天空打来" → 亭柱阴影投向前方，岩面受光面朝悬崖方向
- 锚点是"阴天柔和天光" → 全局柔光，无明显投射阴影方向

### 3.3 权重标记

> 参照规则：`seedance-prompt-order-rules.md` 权重规范标准化章节

锚点光源本身不建议加权——它是"唯一真值"，加权暗示它可能是可选的。如需强化特定光影效果，加权应加在锚点光源派生的具体可视特征上（如 `(肩线发丝金边:1.1)`），而非锚点本身。

---

## 四、跨段落锚点约束

### 4.1 光源方向不可突变

除非场景本身发生了"进入不同光源环境"的事件（如从室外走进室内、从白天转为夜晚），否则锚点光源的方向全程不变。

**突变检测规则：**

```
提取每段的光源方向关键词 → 与首段锚点方向比对
  一致 → 通过
  不一致 → 检查是否有场景切换描述
    有切换 → 通过（但需确认新光源成为新锚点）
    无切换 → 违规，修正
```

### 4.2 时间推进下的锚点演化

如果提示词包含时间推进（黄昏→入夜），锚点光源可以渐变演化，但必须满足：

1. **旧锚点消亡 + 新锚点确立**，不得出现两源共存
2. **过渡段必须描述光源的渐变过程**（如"夕阳色温从暖金渐变为冷紫""天光逐渐取代直射阳光"）

---

## 五、违反锚点规则的判定标准

| 违规类型 | 判定条件 | 动作 |
|----------|---------|------|
| 无锚点 | 全文找不到类型+方向+位置三要素齐全的光源描述 | ❌ 暂停，询问用户设定光源 |
| 锚点残缺 | 有光源描述但缺方向或位置 | ⚠️ 补全后继续 |
| 锚点冲突 | 同一镜头的面光方向与锚点方向矛盾 | ❌ 暂停，修正面光方向 |
| 双锚点 | 开场段落出现两个不相关的主光源（如同一场景同时写"夕阳"和"月光"为主光） | ❌ 暂停，要求用户选择唯一主光源 |
| 锚点漂移 | 无场景切换理由下，中段光源方向与首段不同 | ⚠️ 自动修正为锚点方向，通知用户 |

---

## 六、锚点必须存在——强制检测

**在 S8 质检阶段，如果正文提示词中不存在明确锚点光源，不得继续——必须与用户沟通。**

沟通模板：

> "当前提示词尚未明确设定锚点光源（主光源的类型、方向和位置），这可能导致后续人物面光、环境光影缺乏一致性的推导依据。请确认：这个场景的主光源是什么？从哪个方向、哪个位置打来？例如：「夕阳侧逆光从悬崖外云海上空打来」或「左侧窗格洒落的暖黄烛光」。"

---

## 七、锚点光源规则与其他规则的关系

本规则是"约束层"而非"内容层"，不定义具体的面光写法、骨骼点位、环境光类型。它的唯一作用是：

```
锚点光源 → 确立"唯一圆心"
    ↓
其他规则 → 所有光影描述以此圆心为基准向外扩展
    ├── seedance-closeup-face-lighting-rules.md → 面光从锚点推导
    ├── seedance-prompt-order-rules.md → 光影权重按锚点方向分层
    ├── timeline-execution-rules.md → 环境光照从锚点推导
    └── 任何光影相关描述 → 先对齐锚点再写
```

**本规则不修改任何现有规则文件的内部内容。** 现有规则只需在生成前增加一个前置步骤："当前锚点光源是什么？我的描述是否与它一致？"——回答"是"之后，按自身原有规则继续生成。

## 八、物理合理性校验

> 本校验是锚点光源规则的补充——锚点确立了"圆心在哪"，本校验检查"从圆心画出的光是否合理"。

### 8.1 校验时机

在 S8 质检阶段，锚点存在性确认之后执行。校验不通过时根据等级分别处理：**拦停（红灯）** 或 **警告建议（黄灯）**。

### 8.2 五维校验表

| 维度 | 级别 | 检查内容 | 通过条件 |
|------|------|---------|---------|
| 方向一致性 | 🔴 红灯 | 面光方向 vs 锚点方向，阴影投射方向 vs 光源方向 | 面光来源与锚点同向，阴影投在光源反方向 |
| 强度匹配 | 🟡 黄灯 | 光质软硬 vs 天气/时间/场景 | 阴天→柔光、正午→硬光、黄昏→暖弱光、夜晚→点光源 |
| 色温匹配 | 🟡 黄灯 | 光色 vs 光源类型 | 夕阳→暖金/橙红，月光→冷青/银白，烛光→暖黄，阴天→灰白中性 |
| 空间穿透 | 🟡 黄灯 | 光是否能到达所描述的位置 | 室内光必须从门窗开口进入，不穿透不透明墙壁 |
| 夸张上限 | 🟡 黄灯 | AI 是否过度放大光学效果 | 见 8.3 夸张尺度红线 |

### 8.3 AI 夸张尺度红线

允许 AI 适当放大光影效果以提升画面质感，但不得超过以下边界：

| ✅ 允许（艺术增强） | ❌ 禁止（物理失真） |
|-------------------|---------------------|
| 高光区域扩大至 1.5 倍 | 光穿透不透明物体（如"光透过头骨照亮眼窝"） |
| 明暗对比增强 20-30% | 面部凭空发光、无明显光源却出现定向高光 |
| 光晕柔和向外扩散 | 明暗交界线变成锐利割裂线 |
| 暗部保留隐约细节，不完全死黑 | 逆光镜头却面部全亮（违反光学） |
| 反光适当增强（如金属、水面） | 反射光强度超过主光源 |

### 8.4 常见违规模式与修正

| 违规 | 示例 | 修正 |
|------|------|------|
| 方向矛盾 | 锚点"夕阳侧逆光"→面光"正面柔光均匀铺脸" | 改为"侧后方暖金光线勾勒轮廓，面部柔和半暗" |
| 强度错配 | "阴天薄雾"→"硬朗明暗交界线" | 改为"面部明暗过渡柔和，无明显硬朗交界线" |
| 色温错配 | "冷白月光"→"暖金色光线笼罩" | 改为"冷银月光从窗格洒落"或添加灯笼光源解释暖光来源 |
| 空间穿透 | "封闭石室内无窗"→"侧逆光打亮轮廓" | 改为"微弱烛光从桌面摇曳，面部受暖黄柔光不均匀照亮" |
| 过度夸张 | "光线从眼窝内部向外涌出" | 改为"侧方柔光在眼窝上方形成浅淡高光，下方自然凹陷" |

### 8.5 校验执行流程

```
S8 质检阶段
   ↓
[锚点存在性检测]（§六）
   ↓ 锚点确认
[五维物理校验]
   ├── 🔴 红灯项不通过 → 列出问题 + 给出修正建议 → 等待用户确认
   └── 🟡 黄灯项触发 → 逐条列出 + 标注"建议修正" → 用户可接受或修改
   ↓ 全部通过/确认
继续生成
```

### 8.6 沟通模板

**红灯（拦停）：**

> "检测到光源物理矛盾：[具体问题]。锚点光源为「[锚点描述]」，但第 N 段面部光影写的是「[矛盾描述]」。两处光源方向不一致，光影逻辑无法自洽。建议修改为：[修正方案]。请确认或提供您期望的方向。"

**黄灯（警告）：**

> "检测到光影物理细节与本场景条件不完全匹配：[具体问题]。当前版本仍可生成，但画面可能不够真实。建议修正为：[方案]。需要我直接修改吗？"
