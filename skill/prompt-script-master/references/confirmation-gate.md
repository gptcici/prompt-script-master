# 确认闸门

确认闸门是最高优先级流程规则。任何复述、参考资料建议、动态级别判断、动作方案、经典镜头建议之后，都必须等待用户明确确认，才能进入最终提示词阶段。

## 确认前内容

以下内容全部属于用户沟通层：

- 内容理解与专业复述
- 参考资料识别与缺失资料建议
- 信息完整度判断
- 动态级别判断
- 音乐风格 / 节奏 / 段落确认
- 动作 / 舞蹈 / 表演方案
- 经典镜头与高潮运镜建议
- 选择项、解释、建议和追问

这些内容不能与最终提示词或完整时间轴出现在同一轮回复中。

## 可以视为完整确认的话

只有用户明确表达以下含义，才能进入最终提示词：

- 确认
- 理解正确
- 按这个来
- 采用这个方案
- 可以开始生成提示词
- 进入最终提示词
- 生成最终版
- 就按你说的写

## 不算完整确认的话

以下只算补充或局部确认，不得进入最终提示词：

- 继续
- 你看着办
- 再看看
- 这个可以
- 音乐是流行
- 动作轻一点
- 我上传了参考图
- 这个理解不对
- 用户只回答了一个缺失信息

遇到这些情况，必须更新复述或方案，再次请用户确认。

## 局部确认

用户只确认某一项时，只能锁定该项，例如音乐风格、动态级别、动作方向、镜头方案或参考资料处理方式。局部确认不能自动代表整体确认。

## 完整确认

完整确认必须覆盖：

1. 内容理解正确。
2. 项目类型正确。
3. 信息完整度与缺失项处理方式明确。
4. 动态级别明确。
5. 参考资料处理方式明确。
6. 动作 / 镜头方案如有触发，已确认是否采用。
7. 用户明确允许生成最终提示词。

## 检查要求

如果同一回复中同时出现“内容理解 / 专业复述 / 请确认 / 我建议”等沟通语言，又出现“最终提示词 / 生成规格 / 时间轴”等最终输出结构，视为跳过确认闸门，应当重新拆分流程。
