# 优秀案例示范库规则

## 核心定义

优秀案例示范库不是训练模型，也不是保存用户隐私内容。

它是一组高质量标准案例，用来告诉《提示词脚本大师》：

1. 遇到某类用户输入时，应该如何理解。
2. 应该如何拆解用户需求。
3. 哪些地方需要确认。
4. 哪些地方可以自动默认。
5. 最终提示词应该写成什么结构。
6. 自检合格的提示词应该长什么样。

案例数量不需要一开始很多。第一版先做 5 个高质量案例，比做很多粗糙案例更有用。

## 第一版核心案例

第一版建立 5 个案例：

1. 演唱会英雄镜头
2. 产品广告镜头
3. 情绪人物特写
4. 首尾帧图生视频
5. 错误提示词优化案例

## 每个案例的固定格式

每个案例建议使用：

```text
【案例名称】

【适用场景】
这个案例适合什么类型的用户需求。

【用户原始输入】
用户最开始可能会怎么说。

【系统拆解理解】
系统如何总结用户想表达的内容、主体、场景、情绪和目的。

【初步构思】
系统在生成前如何简单构思镜头方向、空间关系、动作、灯光和节奏。

【需要确认的问题】
如果有关键不确定点，在这里列出。
如果没有严重问题，可以写“无需额外确认，可按专业默认生成”。

【建议参考素材】
建议用户提供哪些素材，例如人物模板图、产品图、首帧、尾帧、分镜图、场景图、风格图。

【最终提示词】
一段可以直接复制粘贴使用的完整中文提示词。

【自检结果】
通过 / 需要优化。

【为什么合格】
简单说明它为什么符合项目规则。
```

## 案例使用方式

案例库只是范本，不是固定答案。

系统遇到相似需求时，可以参考案例的拆解方法和写法，但必须根据当前用户需求重新生成提示词，不能直接照抄。

## 隐私规则

不要把用户真实隐私内容直接放进案例库。

如果要把真实项目做成案例，必须先脱敏。

## 推荐文件结构

```text
examples/concert-hero-shot.md
examples/product-commercial-shot.md
examples/emotional-closeup-shot.md
examples/first-last-frame-video.md
examples/bad-prompt-to-optimized-prompt.md
```

## 最终标准

规则负责告诉系统“应该怎么做”。案例负责告诉系统“做出来应该长什么样”。
